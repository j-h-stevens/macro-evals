# Trace Schema (`data/traces.jsonl`)

One JSON object per line. Generated by `generate_traces.py` with fixed
seed `42`. 5000 traces total across 90 simulated days.

## Top-level fields

| Field | Type | Description |
|---|---|---|
| `trace_id` | string | Zero-padded ID, e.g. `t00001`. Unique. |
| `day` | int | Simulated day 0..89 the trace was opened on. |
| `case_type` | string | Business case classification. One of: `standard_build`, `supplier_substitution`, `regulated_export`, `expedited_delivery`, `custom_configuration`. |
| `events` | array<event> | Ordered list of agent events (see below). |
| `env_signals` | array<string> | Trace-level environment flags present at intake. Subset of `tariff_us_eu`, `import_heavy_sku`, `expedite_flag`, `low_stock_warn`. |
| `outcome` | string | Terminal state: `completed`, `review`, `blocked`, or `failed`. |
| `failure_modes` | array<string> | **Ground truth** failure-mode labels. Empty if clean. Multiple modes may co-occur. |
| `prompt_version` | string | `v1` for day < 60, `v2` for day ≥ 60. Cosmetic wording shift only. |

## Event object

| Field | Type | Description |
|---|---|---|
| `t_ms` | int | Cumulative ms offset from trace start. |
| `agent` | string | One of `orchestrator`, `pricing`, `compliance`, `supply`, `factory`, `scheduling`, `release`. |
| `type` | string | `call`, `return`, `tool`, `handoff`, or `signal`. |
| `to` | string? | Receiving agent (for `call`/`return`/`handoff`). |
| `tool` | string? | Tool name when applicable. |
| `args` | object? | Tool arguments for `call` events. |
| `value` | object? | Return payload for `return` events. |
| `latency_ms` | int | Wall-clock latency attributed to this event. |
| `env_signals` | array? | Per-event signals (rare; usually only the intake event). |
| `note` | string? | Free-text note, used on `signal` and `handoff` events. |

### Notes on `value`

- Pricing returns include `price_usd` and `status`. They also include
  `_raw_price_usd` (underscore-prefixed) — the **pre-drift** price.
  Detection pipelines should ignore underscore-prefixed fields; the field
  exists only so the generator's drift integrity check can be airtight.
- Supply returns include `status` (`in_stock` or `substitute`); when
  `substitute`, also an `alt_part` ID.
- Scheduling returns include `slot_day`.

## Failure mode labels (ground truth)

| Label | Class | Spec prevalence | How it manifests in events |
|---|---|---|---|
| `substitution_reroute` | structural | ~8% | Supply returns `status:substitute`; orchestrator hands off to factory; factory returns `rerouted`. |
| `compliance_skipped_under_tariff` | omission | ~5% | `tariff_us_eu` in `env_signals`, AND no compliance `call`/`return` events exist. (Absence is the signal.) |
| `stale_quote_slow_pricing` | temporal | ~4% | Pricing `return` event has `latency_ms > 3000`. Otherwise events look normal. |
| `pricing_drift` | drift | 100% of traces with `day >= 45` | All pricing returns biased by +0.4%. No single trace looks broken — only population statistics reveal it. |
| `escalation_loop` | structural | ~3% | Compliance → release → compliance cycle occurs ≥ 2 times. |
| `random_noise_failure` | confounder | ~6% | A `signal`-type event with no causal relation to the workflow appears late in the trace. Pipeline noise tests. |

## Confounders (NOT labelled — pipelines should ignore them)

- **Import-heavy SKU** (`import_heavy_sku` env signal): hidden driver
  raising both tariff probability AND substitution probability. Yields
  observed phi(tariff, substitution) ≈ 0.35 without causation.
- **Seasonal volume**: traces/day rises ~30% from day 0 to day 89.
- **Prompt version v2**: at day ≥ 60, status strings shift wording
  (`ok` → `acknowledged`, `quote_ready` → `quote_prepared`,
  `release_ok` → `released`). Semantically identical.
- **Minor noise** (~15% of traces): one event either has a latency
  spike (+400–1200 ms) or a `slot_retry` tool event is appended.
  Unlabelled.

## Determinism

Run `python generate_traces.py` to reproduce byte-identical output. Seed
is hardcoded to 42; all RNG draws funnel through a single `random.Random`.

## Validation that ships with the generator

On every run, `generate_traces.py` prints:
- Total trace count, day span, prompt-version distribution
- Per-failure-mode counts and prevalence vs. spec (±0.5% tolerance)
- Tariff↔substitution phi correlation (target ~0.4)
- Paired drift ratio (drifted ÷ raw) per pricing event:
  pre-day-45 must be exactly 1.0, post-day-45 must be exactly 1.004.
  Any deviation raises `AssertionError`.
- Co-occurrence matrix of failure-mode pairs.
